# Notifications

In-app notifications generated by system events (new chats, messages, feedback replies). Behavior is role-aware: regular users operate on their own notifications; admins operate on the shared admin notification feed.

**Base path:** `/notification` · All endpoints are 🔒 **user**. See the [conventions guide](./README.md) for envelope and errors.

## The notification object

```json
{
  "_id": "665f3d20e3a2a24b8c8d7c01",
  "toId": "665f1c9be3a2a24b8c8d7a01",
  "title": "New message",
  "message": "You have started a new conversation",
  "isRead": false,
  "createdAt": "2026-07-16T11:00:00.000Z",
  "updatedAt": "2026-07-16T11:00:00.000Z"
}
```

Admin-feed notifications have the same shape without `toId`.

---

## Get a notification

```
GET /notification/get-notification
```

- **As a user:** returns the notification identified by `notificationId`.
- **As an admin:** returns the latest admin-feed notification (no id required).

### Query parameters

| Parameter        | Type   | Required                | Description     |
| ---------------- | ------ | ----------------------- | --------------- |
| `notificationId` | string | Users: yes · Admins: no | Notification id |

### Example request

```bash
curl "http://localhost:8001/notification/get-notification?notificationId=665f3d20e3a2a24b8c8d7c01" \
  -H "Authorization: Bearer $ACCESS_TOKEN"
```

### Response `200`

```json
{
  "statusCode": 200,
  "success": true,
  "message": "Notification retrieved",
  "data": { "...": "notification object" }
}
```

### Errors

| Status | Condition                                   |
| ------ | ------------------------------------------- |
| `400`  | Missing `notificationId` (non-admin caller) |
| `404`  | Notification not found                      |

## List notifications

```
GET /notification/get-all-notifications
```

Paginated list — the caller's own notifications for users, the shared admin feed for admins. Supports the standard [pagination/filter/sort parameters](./README.md#pagination-filtering-sorting) (e.g. `?isRead=false`, `?sort=-createdAt`).

### Example request

```bash
curl "http://localhost:8001/notification/get-all-notifications?page=1&limit=10&isRead=false" \
  -H "Authorization: Bearer $ACCESS_TOKEN"
```

### Response `200`

```json
{
  "statusCode": 200,
  "success": true,
  "message": "Notifications retrieved",
  "meta": { "page": 1, "limit": 10, "total": 4, "totalPage": 1 },
  "data": [{ "...": "notification objects" }]
}
```

## Mark read / unread

```
PATCH /notification/update-as-mark-unread
```

Bulk-sets the `isRead` flag on **all** of the caller's notifications (or the whole admin feed for admins).

### Body parameters

| Parameter | Type    | Required | Description       |
| --------- | ------- | -------- | ----------------- |
| `isRead`  | boolean | Yes      | Target read state |

### Example request

```bash
curl -X PATCH http://localhost:8001/notification/update-as-mark-unread \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "isRead": true }'
```

### Response `200`

```json
{
  "statusCode": 200,
  "success": true,
  "message": "Notification updated",
  "data": { "acknowledged": true, "modifiedCount": 4, "matchedCount": 4 }
}
```

### Errors

| Status | Condition                                       |
| ------ | ----------------------------------------------- |
| `400`  | Nothing to update (all already in target state) |

## Delete a notification

```
DELETE /notification/delete-notification
```

Deletes a single notification by id.

### Body parameters

| Parameter        | Type   | Required |
| ---------------- | ------ | -------- |
| `notificationId` | string | Yes      |

### Example request

```bash
curl -X DELETE http://localhost:8001/notification/delete-notification \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "notificationId": "665f3d20e3a2a24b8c8d7c01" }'
```

### Response `200`

```json
{
  "statusCode": 200,
  "success": true,
  "message": "Notification deleted",
  "data": { "...": "deleted notification object" }
}
```

### Errors

| Status | Condition                |
| ------ | ------------------------ |
| `400`  | Missing `notificationId` |
| `404`  | Notification not found   |
